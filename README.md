# Software Development Philosophy

A personal knowledge base capturing my philosophy of software development — built and maintained with [Claude Code](https://claude.ai/code), following [Andrej Karpathy's LLM Wiki pattern](https://karpathy.ai).

## What this is

Over the years I have collected frameworks, ideas, and lessons that shaped how I think about writing software and working with people. This wiki is my attempt to make that thinking explicit, interlinked, and shareable.

The core thesis is simple: **the technical part of software development is the easy part.** The hard part is people — working with yourself, working with others, and working on the right thing.

## Structure

```
wiki/         — markdown pages on concepts, frameworks, and lessons
raw/          — source documents (immutable)
```

Start with [`wiki/index.md`](wiki/index.md) for a full table of contents.

## How it works

Claude Code maintains the wiki. I curate sources, ask questions, and guide the analysis. New ideas get discussed before anything is written, and every factual claim references its source.

Pages are written in Obsidian-flavored markdown with `[[wiki-links]]` between related concepts.
